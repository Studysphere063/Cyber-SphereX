Ethical Hacking Principles

What is Ethical Hacking?

Ethical hacking involves authorized attempts to bypass system security to identify vulnerabilities before malicious hackers can exploit them.

Unauthorized hacking is illegal. Ethical hackers must always have explicit permission.

Certified Ethical Hacker (CEH) Domains

  1. Footprinting and reconnaissance
  2. Scanning networks
  3. Enumeration
  4. Vulnerability analysis
  5. System hacking
  6. Malware threats
  7. Sniffing
  8. Social engineering
  9. Denial-of-service
  10. Session hijacking
  11. Evading IDS, firewalls, and honeypots
  12. Web server hacking
  13. Web application hacking
  14. SQL injection
  15. Wireless network hacking

Hacker Classifications

Type Description Motivation
White Hat Ethical hackers working to improve security Security improvement, legal compliance
Black Hat Malicious hackers breaking laws Financial gain, disruption, theft
Gray Hat Between white and black hats May violate laws but without malicious intent
Script Kiddie Inexperienced using others' tools Thrill-seeking, notoriety

Hacking Terminology

Essential Terms

Zero-day
A vulnerability unknown to the vendor with no patch available
Payload
The part of malware that performs malicious actions
Exploit
Code that takes advantage of a vulnerability
Pivot
Moving from one compromised system to others in the network
Rootkit
Malware designed to hide its existence on a system

Attack Types

  • Phishing - Fraudulent attempts to obtain sensitive information
  • MITM - Man-in-the-middle intercepting communications
  • DDoS - Distributed Denial of Service overwhelming resources
  • SQLi - SQL Injection manipulating database queries
  • XSS - Cross-site scripting injecting client-side scripts

Legal Framework

Key Cybersecurity Laws

  • Computer Fraud and Abuse Act (CFAA) - Primary US anti-hacking law
  • General Data Protection Regulation (GDPR) - EU data protection
  • Computer Misuse Act - UK legislation against unauthorized access
  • DMCA - Digital Millennium Copyright Act

Penalties for Unauthorized Access

Unauthorized access to computer systems can result in:

  • Substantial fines (up to millions of dollars)
  • Imprisonment (years in federal prison)
  • Civil lawsuits
  • Permanent criminal record

Bug Bounty Programs

Legal way to find and report vulnerabilities for rewards:

  • HackerOne
  • Bugcrowd
  • Company-specific programs (Google, Microsoft, etc.)

Nmap Network Scanning

Common Nmap Commands

# Basic scan
nmap -sV 192.168.1.1

# Aggressive scan
nmap -A -T4 target.com

# Port range scan
nmap -p 1-1000 10.0.0.1

# OS detection
nmap -O 192.168.1.100

# Save output
nmap -oN scan.txt -oX scan.xml target.com

Nmap Scripting Engine

# Vulnerability scanning
nmap --script vuln target.com

# HTTP enumeration
nmap --script http-enum -p80 target.com

# Brute force attacks
nmap --script ssh-brute --script-args userdb=users.txt,passdb=passwords.txt target.com

Cryptography Fundamentals

Encryption Types

  • Symmetric (AES, DES) - Same key encrypts/decrypts
  • Asymmetric (RSA, ECC) - Public/private key pairs
  • Hashing (SHA, MD5) - One-way functions

Common Attacks

  1. Brute force
  2. Dictionary attacks
  3. Rainbow tables
  4. Side-channel attacks
  5. Man-in-the-middle

Python Crypto Example

from cryptography.fernet import Fernet

# Generate key
key = Fernet.generate_key()
cipher = Fernet(key)

# Encrypt
message = "Secret message".encode()
encrypted = cipher.encrypt(message)

# Decrypt
decrypted = cipher.decrypt(encrypted)
print(decrypted.decode())
